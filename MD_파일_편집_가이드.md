# MD 파일 편집 가이드

## MD 파일이란?

MD 파일은 **Markdown** 형식의 문서 파일입니다. 텍스트 기반으로 작성되며, 표, 제목, 리스트 등을 쉽게 작성할 수 있습니다.

## MD 파일 편집 방법

### 방법 1: Cursor (현재 사용 중인 에디터) - **권장**

1. **Cursor에서 직접 열기**
   - 파일 탐색기에서 `메뉴_리스트_정리.md` 파일을 더블클릭
   - 또는 Cursor에서 `File` → `Open File...` → `메뉴_리스트_정리.md` 선택

2. **편집하기**
   - 일반 텍스트처럼 편집 가능
   - 표 형식은 그대로 유지하면서 내용만 수정
   - 저장: `Ctrl + S`

3. **미리보기**
   - Cursor에서 Markdown 미리보기 기능 사용 가능
   - 우측 상단의 미리보기 아이콘 클릭

### 방법 2: 메모장 (Notepad)

1. `메뉴_리스트_정리.md` 파일을 우클릭
2. "연결 프로그램" → "메모장" 선택
3. 편집 후 저장 (`Ctrl + S`)

**주의**: 메모장은 표 형식이 깨질 수 있으므로 주의가 필요합니다.

### 방법 3: VS Code

1. VS Code 설치
2. `메뉴_리스트_정리.md` 파일을 VS Code로 열기
3. 편집 후 저장

### 방법 4: 온라인 에디터

- **GitHub**: GitHub 저장소에서 직접 편집 가능
- **StackEdit**: https://stackedit.io/
- **Dillinger**: https://dillinger.io/

## 변경사항을 GitHub에 반영하는 방법

### 방법 1: 자동 커밋 스크립트 사용 (가장 간단) - **권장**

1. **파일 편집 후 저장**
   ```
   메뉴_리스트_정리.md 파일을 편집하고 저장
   ```

2. **PowerShell에서 실행**
   ```powershell
   .\auto-commit.ps1 "메뉴 리스트 업데이트"
   ```

   또는 배치 파일 사용:
   ```cmd
   auto-commit.bat
   ```

3. **완료!**
   - 자동으로 Git에 커밋되고 GitHub에 푸시됩니다.

### 방법 2: 수동으로 Git 명령어 사용

1. **파일 편집 후 저장**

2. **PowerShell 또는 Git Bash에서 실행**
   ```bash
   # 변경사항 확인
   git status
   
   # 변경사항 추가
   git add 메뉴_리스트_정리.md
   
   # 커밋
   git commit -m "메뉴 리스트 업데이트"
   
   # GitHub에 푸시
   git push origin main
   ```

### 방법 3: GitHub 웹에서 직접 편집

1. GitHub 저장소 페이지 접속
2. `메뉴_리스트_정리.md` 파일 클릭
3. 우측 상단의 연필 아이콘(✏️) 클릭
4. 편집 후 하단의 "Commit changes" 클릭

## 자동 동기화 설정

### 파일 변경 감지 자동 업데이트 (선택사항)

파일을 저장할 때마다 자동으로 GitHub에 반영하려면:

```powershell
.\watch-and-sync.ps1
```

이 스크립트를 실행하면:
- 파일 변경을 자동으로 감지
- 변경 후 10초 대기 (연속 변경 방지)
- 자동으로 커밋 및 푸시

**종료**: `Ctrl + C`

## MD 파일 편집 팁

### 표 편집하기

표는 다음과 같은 형식입니다:

```markdown
| 번호 | 메인 메뉴 | 기본값 | 서브 선택 리스트 | 함수명 |
|------|----------|--------|------------------|--------|
| 1 | 스타일 | 기본형 | 기본형, ㅁ다리 | `setMium(0)` |
```

- 각 셀은 `|`로 구분됩니다
- 헤더와 본문 사이에 `|------|` 형식의 구분선이 필요합니다
- 셀 내용을 수정할 때는 `|` 사이의 텍스트만 변경하면 됩니다

### 새 행 추가하기

표에 새 행을 추가하려면:

```markdown
| 47 | 새 메뉴 | 기본값 | 옵션1, 옵션2 | `setNewMenu()` |
```

기존 행을 복사해서 수정하면 쉽습니다.

### 제목 추가하기

```markdown
## 제목 (2단계)
### 소제목 (3단계)
#### 더 작은 제목 (4단계)
```

## 주의사항

1. **표 형식 유지**: 표를 편집할 때 `|` 구분자를 반드시 유지해야 합니다
2. **인코딩**: 파일은 UTF-8 인코딩으로 저장되어야 합니다 (한글 지원)
3. **줄바꿈**: 표 내에서 줄바꿈은 `<br>` 태그를 사용합니다
4. **백업**: 중요한 변경 전에는 백업을 권장합니다

## 문제 해결

### 표가 깨져 보이는 경우
- 각 행의 `|` 개수가 동일한지 확인
- 헤더와 본문 사이의 구분선이 있는지 확인

### 한글이 깨지는 경우
- 파일 인코딩을 UTF-8로 저장
- Cursor나 VS Code 사용 권장

### Git 푸시가 안 되는 경우
- 인터넷 연결 확인
- GitHub 인증 확인
- `git pull` 먼저 실행 후 다시 시도
